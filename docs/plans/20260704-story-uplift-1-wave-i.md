# Story uplift 1/5 — волна I: модули 01–03 (Акт I «Тебе объясняют»)

## Overview

Второй план семейства story-uplift (см. `docs/plans/20260704-story-uplift-0-pilot.md`
и архив `docs/plans/completed/20260703-story-uplift.md`). Раскатка сюжетного
слоя на модули 01, 02 (кроме 02-04 и 02-06 — сделаны в пилоте) и 03 —
15 юнитов Акта I: наставники объясняют, читатель осваивается.

**Предусловие запуска:** план `0-pilot` завершён, владелец принял тон пилота
(кринж-тест) и внёс ретро-правки канона. Запуск этого плана = подтверждение
приёмки пилота.

Ключевые биты волны (детали — в записях карты сцен): первое личное появление
Виктора (01-01) и Стаса (01-04), первая «шапка Зои» (02-03), флешбэк Дани
(03-03), расплата Стаса за NULL (03-06).

## Context

- Канон истории: `docs/story-canon.md` — читается целиком перед каждой правкой
  прозы (реестр голосов §2, слоты §4.1, лестница §4.3, гигиена §5, биты §6,
  реестр нитей §7, эталоны §9).
- Карта сцен: `docs/story-scene-map.md` — запись юнита (форма, участники,
  pitch, нити, гейт-заметки) = обязательный вход каждой юнит-задачи; там же
  бюджет-заметка модуля перед его юнитами.
- Гейты сцен: `.claude/skills/lecture-writer/references/checklist.md`
  (внесены планом `0-pilot`).
- Модули волны: `lectures/01-data-types/`, `lectures/02-schema-and-constraints/`,
  `lectures/03-crud-fluency/` — правятся только `i18n/{ru,en}/README.md`.
- Предусловия окружения: команды из корня репозитория; `pnpm install`
  выполнен. Docker-песочница не нужна — код и stdout не трогаются.

## Development Approach

- Одна задача = один юнит (RU+EN вместе), полностью до перехода дальше;
  внутри волны — по номерам.
- Правки — только `i18n/{ru,en}/README.md`. Код юнитов, `query.sql`,
  `sqlc.yaml`, `Makefile` и секции `## Запуск` / `## Running it` неприкасаемы.
- Полные сцены — только у юнитов со ★ по карте; остальным — форма по лестнице
  §4.3 из записи карты. Сцена **заменяет** прозу, которую драматизирует
  (cap ×1.5 по словам), и обязана быть сильнее заменяемого открытия — иначе
  форма ниже по лестнице (правило отката).
- Роль тестов играют гейты (см. Testing Strategy); зелёные — до следующей
  задачи. CRITICAL: при отклонении объёма обновлять этот план (➕/⚠️).
- Коммит на задачу: `docs:` — EN subject, RU body (формат по CLAUDE.md).

## Testing Strategy

- После каждого юнита — «линтер юнита» (команда в Technical Details): ноль
  ошибок, предупреждения гендер-эвристики разобраны перечитыванием реплик.
- Сайт-гейты `make web-check-coverage` / `make web-build` — на чеке волны
  (финальная задача), не на каждом юните.
- Консистенси-чек волны: греп нитей §7, сверка голосов §2, полный линтер по
  трём модулям.

## Progress Tracking

- При расхождении с планом — править план в том же коммите.

## Technical Details

- «Линтер юнита» = `node .claude/skills/lecture-writer/scripts/check_unit.mjs
  <путь-юнита> --against-head` (запуск из корня; пути юнитов — в блоке Files
  задачи).
- Порядок работ юнит-задачи: канон целиком → запись юнита в карте сцен →
  текущие RU/EN README → RU-правка → EN-зеркало → линтер юнита → diff-скоуп.
- EN-реплики читателя — по решению, зафиксированному в Task 1 (ретро пилота);
  EN-эталоны сцен — §9 канона.
- Тон Акта I: наставники объясняют, читатель спрашивает; катехизис Марата
  экономно (эхо-точки — только по реестру §7).
