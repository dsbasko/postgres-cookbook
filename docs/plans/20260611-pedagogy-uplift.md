# План доработки лекций: плавный рост новичок → профи

Источник: разбор курса Яндекс Практикума «Go-разработчик расширенный» (8 спринтов,
130 уроков теории — выгружены в `temp/yandex/`) против наших 63 юнитов. Полные
материалы анализа — `temp/ai/pedagogy/` (методичка приёмов Практикума
`yandex-playbook.md`, карта проблем курса `ours-gaps.md`, по-модульные отчёты).

## Вердикт: гипотеза подтвердилась, но причина — не та, что кажется

Интуиция «недостаточно плавно и недостаточно глубоко» верна, но за ней стоят
**три разные** проблемы, и самая крупная в постановке задачи даже не названа.

1. **Читатель пассивен все 63 юнита.** На весь курс — ноль вопросов
   самопроверки и два микро-задания («попробуй `\timing`» в 00-03, «проверь
   арифметику» в 08-01). Учащийся только читает и запускает готовое, причём
   вывод уже вклеен в README — запускать необязательно. Это главный контраст с
   Практикумом, где retrieval practice — несущая конструкция, а инфраструктура у
   нас под неё идеальная (детерминированный stdout = готовый оракул для
   «предскажи и проверь»). Рука не ставится вовсе.

2. **Плавность: термины «в кредит».** Системный паттерн — понятие используется
   до введения и без глоссы: пул (00-01), SQLSTATE (весь модуль 02), WAL
   (02-03), планировщик (04-02), `::` и `coalesce`/`sum` (01-01), `sqlc.arg()`
   (03-01), `UNION ALL` (00-01 → несущая в 08-04). Базовый `SELECT` как таковой
   не преподаётся: 03-02 даёт `WHERE/ORDER/LIMIT` одним абзацем и сразу уходит в
   keyset. Бьёт сильнее всего по новичку в первых модулях.

3. **Глубина: провалы точечные, не повальные.** Курс глубок там, где речь о
   «когда НЕ применять» (заборчики, таблицы-развилки — это сила). Но
   отсутствуют конкретные, самые дорогие вещи: `LEFT JOIN` + условие по правой
   таблице в `WHERE` → тихий `INNER` (баг №1 прикладника, в курсе нет вовсе);
   fan-out и задвоение агрегатов на стыке 04-02 → 04-03; чтение `EXPLAIN` без
   статистики и `loops`; оговорка к `SERIALIZABLE` (защита работает, только если
   ВСЕ транзакции инварианта идут под ним); оптимистичная блокировка в модуле 05.

Отдельно: две **фактические ошибки**, которые надо чинить независимо от плана —
02-06 (таблица `ALTER`: `ADD COLUMN … NOT NULL` с константным `DEFAULT`
мгновенен с PG11, сканирует таблицу `ALTER COLUMN … SET NOT NULL`) и 00-04
(модель `$1` описана как generic plan; в дефолтном режиме Postgres планирует
после bind, зная значения).

## Чего у Практикума брать НЕ надо

> **⚠️ SUPERSEDED (2026-07-04).** Абзац ниже про «не добавляем персонажей»
> отменён ратификацией канона истории — см. `docs/story-canon.md` (§10) и план
> `docs/plans/20260704-story-uplift-0-pilot.md`: курс получает сквозную историю
> команды Brew с персонажами и сценами. Остальные выводы раздела (без GIF,
> без эмоциональных рамок, без платформенных виджетов) остаются в силе.

Главный вывод методички: Практикум сам **демонтирует** ситком по мере роста
ученика. В спринтах 1–4 — пять персонажей и сюжетные арки; в 7–10 реплик нет
вообще, их функции (вопрос ученика, снятие страха, сократика) переданы дешёвым
заменителям (голос «мы», самовопрос автора, точечные персонажи-аналогии только
на пике абстракции). Наш курс — прозовый, без персонажей, и это **правильно**
для продвинутой аудитории. Не добавляем персонажей. Добавляем их функции.

Так же не тащим: GIF и анимации (заменяем парой статичных схем «до/после»),
полные эмоциональные рамки («Гордимся вами!» без живой команды звучит фальшиво),
кликабельные реплики и платформенные виджеты.

## Техническое ограничение движка (решающее для дизайна)

`@dsbasko/cookbook-engine@1.1.0` рендерит markdown через `remark-gfm` +
`remark-github-blockquote-alert` + `rehype-pretty-code`. `rehype-raw` в сборке
**нет** — значит, сырой HTML (`<details><summary>`) сейчас не отрендерится.

Из этого следует развилка для интерактива (квизы, спойлеры подсказок):

- **Доступно сегодня, без правок движка:** GitHub-алерты
  `> [!NOTE] / [!TIP] / [!IMPORTANT] / [!WARNING] / [!CAUTION]` — готовый формат
  каллаутов. Самопроверку можно собрать в «инвертированном» виде: вопрос прозой
  → ответ в алерте ниже (раскрытия по клику нет, но блок читается как
  мини-конспект — аналитики отметили, что фидбек Практикума и так самодостаточен).
- **Требует фичи движка** (новый remark-directive или `rehype-raw`):
  сворачиваемый спойлер и квиз-компонент с пофрагментным фидбеком и
  двухступенчатой подсказкой. По правилу проекта это **бамп версии engine**, а
  не правка `web/`.

## План доработки по тиерам

### Тиер 0 — без правок движка, начинаем сразу

Переносится буквально, инфраструктура уже есть.

- **Глосса на первом употреблении.** Для каждого термина «в кредит» — одна
  фраза-определение или ссылка на юнит, где он раскрыт. Список целей —
  `ours-gaps.md` §1.7. Это снимает большую часть проблемы плавности.
- **Строка пререквизитов в начале юнита.** «Опирается на: 03-03 (транзакции),
  04-01 (LEFT JOIN)». Активация прошлого знания, которой сейчас нет (мосты живут
  только в конце предыдущего юнита).
- **Чинить точечные провалы глубины** — топ-10 из `ours-gaps.md` §3, начиная с
  `LEFT JOIN`+`WHERE`→`INNER` и fan-out (это семантика, а не перформанс — в
  модуль 06 не откладывается).
- **Чинить 2 фактические ошибки** (02-06, 00-04) — это баги.
- **Каллауты через алерты** вместо части прозы: ловушка → `[!WARNING]`,
  трейд-офф → `[!TIP]`. Дозировка из методички §3: проза дольше 5 абзацев без
  кода/таблицы/вопроса — сигнал переписать.
- **Самопроверка в инвертированном виде** — хотя бы одна точка на ~15 минут
  чтения: «Сколько строк вернёт запрос? Какой SQLSTATE?» → ответ в `[!NOTE]`
  ниже. Это первый шаг к активной роли без правок движка.

### Тиер 1 — высокий рычаг, нужна фича движка

- **Квиз-компонент с пофрагментным фидбеком** (по образцу приёма №5 Практикума —
  самого частотного в их курсе) и **двухступенчатой подсказкой** (намёк →
  решение). Разбор на каждый вариант, включая неверные; дистракторы целятся в
  реальные заблуждения. Готовые таблицы курса (уровни изоляции 05-04, матрица
  2×2 в 08-06, карта ACID 05-01) сами просятся в формат «какая аномалия — какой
  уровень её поймает».
- Реализация: remark-directive `:::quiz` / `:::spoiler` в движке (предпочтительно
  — типобезопаснее `rehype-raw`), затем бамп пина в `web/`.
- Кандидаты второго прохода (глубокое чтение всех 12 спринтов, включая
  алгоритмические 13–16; дельта в `temp/ai/pedagogy/style-delta.json`):
  сворачиваемый спойлер для ответов самопроверки (предсказание сейчас частично
  обесценено мгновенно видимым ответом); якорь/подсветка конкретной строки
  листинга, на которую указывает проза; табы `query.sql` ↔ сгенерированный
  sqlc-Go — только генерацией из исходников юнита (рукотворные параллельные
  листинги у Практикума гниют); рендер формул для редких количественных
  выкладок (селективность, вероятность коллизий).

### Тиер 2 — структура курса

- **Урок-резюме на модуль** (чекпоинт 5 минут, не абзац в хвосте): карта
  «инвариант → констрейнт → SQLSTATE» (02), чек-лист ловушек JOIN (04), дерево
  «какой индекс выбрать» (06).
- **Карта прогресса** (приём №7): одна схема навыков поверх Brew в итогах
  модуля, пройденное закрашено, будущее серым. Меняется только раскраска.
- **Мини-капстоны модулей** — один запрос/фича, собирающая инструменты модуля
  (04 — один запрос со всеми JOIN; 07 — поиск по меню Brew; 08 — окно + LATERAL).
- **Жанр «типичные ошибки ученика»** рядом с заборчиком: 25P02 после ошибки в
  `BEGIN`, `= NULL` в `WHERE`, забытый `rows.Close()`, целочисленное деление в
  деньгах.

### Не входит в план (сильные стороны — не трогать)

Холодное открытие болью Brew, сквозные якоря (Карина, заказ #1), детерминизм как
учебный материал, заборчики, таблицы-развилки «когда НЕ брать», ASCII-схемы,
Go-линия («в Go это X», эффект `LEFT JOIN` на nullable sqlc), мосты с номерами
юнитов и выплата старых обещаний, дуга 40001 на три юнита. Это лучшее в курсе.

## Чек-лист урока (рабочий артефакт)

1. Открытие: боль / измерение / мост от пройденного + строка пререквизитов.
2. Одна концепция = одна главка: проблема → «когда использовать» → код → вывод →
   trade-off → проверка.
3. Ни одного листинга без вывода; ни одного вывода без комментария.
4. Сломанная версия раньше правильной; SQLSTATE и точная ошибка — материал, а не
   сноска.
5. Хотя бы одна самопроверка на ~15 минут чтения.
6. Каждому инструменту — «когда НЕ использовать» (уже есть в заборчиках).
7. Проза дольше 5 абзацев без кода/таблицы/вопроса — переписать.
8. Закрытие: «теперь ты можешь…» зеркально целям → мост одной фразой.
9. Глосса на первом употреблении каждого нового термина.
10. В итогах модуля — карта прогресса.

3. Решение по Тиер 1 (фича движка) — после пилота, на реальном объёме квизов.
