# План: вводный on-ramp в module 00

> **Статус.** Реализовано (вариант A), кроме одного: `make run` нового юнита не
> прогнан здесь — sandbox порт 5432 занят посторонним Postgres. Гейты пройдены:
> `make build` (47 модулей), `make gen` идемпотентен, `web-check-coverage`
> 63/63 RU+EN, повисших ссылок нет, таблица `## Запуск` байт-точна (tabwriter
> сверен автономно). Остаётся: пользователь прогоняет `make run` и подтверждает
> хвост строки версии. Канон (`docs/course-canon.md` §1) — источник нарратива.

## Проблема

История курса (линза «Первый год бэкендером в Brew», герой, рост Brew, якорь
заказ #1, финал = `PUBLICATION` → `kafka-cookbook`, карта 11 модулей) уже
спроектирована в каноне §1 и описана в корневом `README.md` (EN). Но **ученик
её не видит**: на сайте/в уроке он попадает прямо в 00-01, который с третьей
строки уходит в client-server теорию. Нет входной точки, рассказывающей историю
до первой технической темы. Отсюда «с места в карьер».

## Решение

Новый первый урок **`00-01-meet-brew` «Знакомство с Brew»**: история + герой +
что построим + карта тем + мягкое демо «первый контакт» (версия сервера + перепись
мира Brew в цифрах). Существующие 00-01..00-05 сдвигаются в 00-02..00-06 с лёгкой
правкой холодных открытий для непрерывности. История дублируется на уровень курса
(course.yaml + описание модуля + RU-нарратив).

## Демо нового юнита (sqlc, детерминированное)

- `query.sql`: `ServerVersion :one` (`SELECT version()`) + `BrewWorld :many`
  (9-way `UNION ALL` с фиксированным `ord`, `ORDER BY ord`).
- Счётчики из seed (зафиксированы): customers 3, drinks 5, articles 2, orders 3,
  outbox 0, processed_outbox_ids 0, shops 2, order_items 4, inventory 5. Итого 24.
- Детерминизм: версия с оговоркой про хвост (как в текущем 00-01); счётчики
  фиксированы seed'ом; `now()`/uuid/random не печатаем. Амаунты не печатаем (§9.7).
- `schema.sql` пустой (только канон). `sqlc.yaml` — копия эталона, `out: internal/db`.

## Радиус правок (перенумерация +1)

1. **5 каталогов** — `git mv`; в каждом: go.mod `module`-строка, `cmd/demo/main.go`
   импорт `…/00-0X-slug/internal/db`, строка в `go.work`. (`sqlc.yaml`/`schema.sql`
   путь не меняется — глубина та же.)
2. **Внутримодульные ссылки по номеру в прозе** READMEs (ru+en) 00-02..00-06 —
   remap `01→02 … 05→06` (заголовки, «дальше — 00-0X», «придём в 00-0X»).
3. **Межмодульные ссылки по номеру** (найдены grep'ом): `09-01` (→ старый 00-03,
   станет 00-04), `10-04` (→ старый 00-05, станет 00-06), `03-05` (escape-hatch

4. **course.yaml** — добавить lesson `00-01-meet-brew` (duration 15m, tags
   [intro, brew, roadmap]); существующие 5 слагов → +1; описание модуля 00

5. **course-canon.md** — Приложение A: добавить строку нового юнита, переименовать

## Сшивка холодных открытий 00-02..00-06 (лёгкая)

Пара строк в начале каждого, чтобы сюжет был непрерывным: 00-02 — «теперь, когда
видел карту, первый технический шаг…». Остальные — поправить только номера-ссылки
и хвостовой крючок предыдущего юнита.
